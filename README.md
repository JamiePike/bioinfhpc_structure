# Cookiecutter for HPC bioinformatics projects

Cookiecutter based structure for setting up a bioinformatics project on an HPC. Often HPC will split work across project specific data and scratch directories, where data directories are regularly snapshot, but scratch are not. Ensuring that project are effectvely managed across these different directories can be tricky. 

This cookiecutter template aims to establish a project folder strcuture roughly outlined by [Nobel (2009)](#references) 


```
.
├── CITATION.cff            # project citation metadata (inc. ORCID)
├── LICENSE
├── README.md               # Main project documentation and pipeline execution guide
├── config/                 # Project-specific runtime configurations
│   ├── samples.tsv         # Experimental design matrix (sample IDs and metadata)
│   └── slurm_profile/      # Slurm scheduler execution directives
│       └── config.yaml     # Cluster resource allocation limits (partitions, RAM, time)
├── data/                   # Directory tracking input datasets (managed via scratch link)
│   ├── external/           # Public or third-party datasets (e.g., dbSNP, cosmic)
│   ├── raw/                # Immutable, un-edited starting files (e.g., raw FASTQ)
│   └── reference/          # Reference genomes, transcripts, and index files (e.g., FASTA)
├── doc/                    # Documentation and prose logs
│   └── notebooks/          # Digital lab notebooks for exploratory analysis and plotting
│       ├── template_jupyter.ipynb  # Starter Jupyter Notebook structure
│       ├── template_quarto.qmd     # Starter Quarto Markdown structure (Typst PDFs)
│       └── template_rmarkdown.Rmd  # Starter R Markdown structure
├── project_scratch         # Gateway symlink pointing directly to high-speed HPC storage
├── results/                # Organized analytical outputs generated by the workflow
│   └── figures/            # Publication-ready plots harvested from downstream notebooks
├── src/                    # Monolithic, production-ready source code and helper scripts
└── workflow/               # Pure pipeline execution engine (isolated from data/configs)
    ├── Snakefile           # Main Snakemake entry point and workflow orchestrator
    ├── containers/         # Recipes for immutable container images (Singularity/Apptainer)
    ├── envs/               # Version-controlled software dependency environments (Conda YAML)
    └── rules/              # Modularized Snakemake rule files included by the main Snakefile
```

## References
