# Cookiecutter for HPC bioinformatics projects

[Cookiecutter](https://github.com/cookiecutter/cookiecutter) template for setting up a bioinformatics project on an HPC. 

To implement, install [Cookiecutter](https://github.com/cookiecutter/cookiecutter)  (with `pip`, `brew`, or `conda`)

 Then, initilaise the project. 

```
cookiecutter https://github.com/JamiePike/bioinfhpc_structure.git
```

Often HPC will split work across project specific data and scratch directories, where data directories are regularly snapshot, but scratch are not. Ensuring that project are effectvely managed across these different directories can be tricky. 

This cookiecutter template aims to establish a project folder strcuture roughly outlined by [Nobel (2009)](#references) 


```
data_directory
└── {{cookiecutter.project_slug}}
    ├── .gitignore              # .gitignore to prevent upload of project scratch etc to GitHub
    ├── CITATION.cff            # project citation metadata (inc. ORCID)
    ├── LICENSE
    ├── README.md               # summary project documentation and/or pipeline execution guide
    ├── config/                 # project-specific runtime configurations
    │   └── samples.tsv      # SLURM scheduler execution directives
    │      
    ├── data/                   # directory tracking input datasets 
    │   ├── external/           # public or third-party datasets 
    │   │   └── README.md
    │   ├── raw/                # immutable, un-edited starting files (e.g., raw FASTQ)
    │   │   └── README.md
    │   └── reference/          # reference genomes, transcripts, and index files (e.g., FASTA)
    │       └── README.md
    ├── doc/                    # documentation and prose logs
    │   ├── references.bib      # references bibtext file
    │   ├── manuscript          # manuscript draft or links to it  
    │   └── notebooks/          # digital lab notebooks for exploratory analysis and plotting
    │       ├── template_jupyter.ipynb  # starter Jupyter Notebook structure
    │       ├── template_quarto.qmd     # starter Quarto Markdown structure (Typst PDFs)
    │       └── template_rmarkdown.Rmd  # starter R Markdown structure
    ├── logs                    # directory for logs
    ├── project_scratch -> [path/to/scratch] # symlink pointing directly to HPC scratch work area for large file storage
    │   └── tmp                 # tmp folder (placeholder)    
    ├── results/                # analytical outputs generated by the workflow
    │   └── figures/            # publication-ready plots from notebooks
    ├── src/                    # source code and helper scripts
    └── workflow/               # pipeline execution engine 
        ├── Snakefile           # main Snakemake file
        │   └── profiles/      # SLURM scheduler execution directives
        │       └── deafault/
        │           └── profile.yaml  # cluster resource allocation limits (partitions, RAM, time)
        ├── containers/         # Rrcipes for container images (Singularity/Apptainer)
        ├── envs/               # version-controlled software dependency environments (Conda YAML)
        └── rules/              # snakemake rule files included by the main Snakefile
```
